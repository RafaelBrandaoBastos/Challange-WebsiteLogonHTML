Page: https://subtle-zuccutto-4b6f19.netlify.app/


História:
O cliente busca uma solução digital para facilitar a inscrição de pessoas no seu programa de estágios, para isso necessita de um cadastro com algumas informações básicas e senha para um futuro login.

Cenário:
O PO deseja que sejam entregues duas telas conforme exemplificado na plataforma do Figma, seguindo padrões de responsividades e uma boa experiência de usuário.

Requisitos:
Desenvolvimento de acordo com a criação dos designers no figma.

Requisitos técnicos (obrigatórios):
→ Todo site (inclusive feedback de validações) em inglês.
→ Responsividade (uso de flexbox/grid e media queries).
→ Apenas utilização de CSS puro, não sendo permitido o uso de frameworks como bootstrap.
→ Organização de código (Html, CSS e JS) e utilização de boas práticas.
→ Manter código versionado em repositório no GitHub (utilização de branchs de funcionalidade e commits semânticos).
→ Variáveis e commits em inglês.
→ Media Querie principal width: 650px
→ Desenvolver todo o site no modelo SPA (Single-Page Application)

Extras:
Validação de navegação (botão register) com base nos dados preenchidos (inputs e checkbox)
Armazenar e persistir dados preenchidos (local storage)
Validação dos campos com JavaScript (feita com Regex)
Validações:
→ Nome: mais de 4 dígitos e um espaço
→ E-mail: permitido apenas letras minúsculas e sem assentos, números,
ponto '.' e @
→ Senha: apenas números, maior ou igual a 6 e menor de 9. (utilizar mask)
→ Data em date Picker
→ Telefone: 11 dígitos necessariamente e não permitido caracteres especiais ou letras.
→ Checkbox: os termos deve ser aceitos. Apenas poderá ser acessada a pagina de sucesso se todos os campos estiverem dentro dos parâmetros
Fazer o deploy no github do projeto.
